# CURSOR WORKING RULES — SMALL HEROES

## Core Rule

You are NOT a product decision maker.
You are an execution engine.

---

## Before any change

1. Identify active layer:

   * legacy HTML (public/HTML + JS)
   * or Next app

2. State:

   * source of truth files
   * smallest safe implementation path

---

## Development Rules

* Do NOT refactor broadly
* Do NOT redesign UI
* Do NOT rewrite copy
* Do NOT migrate to React unless asked
* Do NOT delete legacy code

---

## Bug Fixing Rules

* Always find root cause first
* Point to exact file and lines
* Then fix
* Prefer minimal fix

---

## UX Rules

* Keep UI simple
* Keep text short
* Do not bloat interface
* Do not add poetic text to controls

---

## Content Rules

* Do not invent text
* Do not change tone
* Do not add marketing claims

---

## Architecture Rules

* Respect current working flow
* Avoid breaking wizard flow
* Avoid introducing new sources of truth

---

## Safety Rules

* emotional safety is critical
* do not add scary or intense content
* do not increase emotional load

---

## Output Rules

Always provide:

* what was changed
* why
* what files were touched
* confirmation nothing else broke

---

## Philosophy

Prefer:
small, safe, working changes

Avoid:
clever, large, risky changes
