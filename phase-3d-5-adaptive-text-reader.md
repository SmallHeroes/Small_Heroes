# Phase 3d.5 — Adaptive text color + reader polish

## Context
Text on dark backgrounds (night scenes) is unreadable — dark text on dark image. The gradient overlays create visible light/dark bands that look like artifacts. End screen only has "back to home" — needs "read again". The lighting mode per page is already generated by the storyboard but not persisted or used by the reader.

## Tasks

### T1 — Persist `lighting` on BookPage + derive text color scheme

The storyboard already generates `lighting` per page (one of: `soft_daylight`, `warm_golden`, `moonlit`, `indoor_warm`, `dramatic_soft`). Like textZone before, it's used for prompts but thrown away.

**Step 1: Add columns to BookPage**

In `backend/schema.prisma`, add to `BookPage`:
```prisma
model BookPage {
  // ... existing fields ...
  textZone    String?
  lighting    String?   // 'soft_daylight' | 'warm_golden' | 'moonlit' | 'indoor_warm' | 'dramatic_soft'
}
```

Run `npx prisma db push` then `npx prisma generate`.

**Step 2: Persist lighting during generation**

Wherever `textZone` is now persisted on BookPage creation (this was wired in 3e.3), add `lighting: pageStoryboard.lighting` alongside it. Search for where `textZone` is set on BookPage — `lighting` comes from the same storyboard object.

**Step 3: Return lighting from orders API**

In `app/api/orders/[orderId]/route.ts`, add `lighting: true` to the page select (next to `textZone`), and add `lighting: p.lighting ?? null` to the contentPages map.

**Step 4: Derive textColorScheme in reader**

In `reader-v2.tsx`, add a helper to derive the color scheme from lighting:

```typescript
type TextColorScheme = 'light' | 'dark';

function deriveTextColorScheme(lighting: string | null | undefined): TextColorScheme {
  // Dark backgrounds need light text
  const darkLightingModes = ['moonlit', 'dramatic_soft'];
  if (lighting && darkLightingModes.includes(lighting)) return 'light';
  return 'dark';
}
```

Add `lighting` to the `BookPage` type:
```typescript
type BookPage = {
  // ... existing fields ...
  lighting?: string | null;
};
```

Add `textColorScheme` to `ReaderPage`:
```typescript
type ReaderPage = {
  // ... existing fields ...
  textColorScheme: TextColorScheme;
};
```

In `normalizeReaderPages`, compute it:
```typescript
return {
  // ... existing fields ...
  textColorScheme: deriveTextColorScheme(page.lighting),
};
```

### T2 — Adaptive text color + backdrop in CSS

Replace the gradient-based overlay with a clean semi-transparent backdrop behind the text block itself.

**In `reader-v2.module.css`:**

Remove the `.overlayTextGradient` element styling entirely (lines 207-212 and the zone-specific gradient rules at lines 247-249, 261-263).

Add two color scheme classes:

```css
/* ── Dark text on light backgrounds (default) ── */
.textSchemeDark .overlayPageText {
  color: #2a241a;
  text-shadow: 0 1px 3px rgba(255, 255, 255, 0.4);
}

.textSchemeDark .overlayTextBackdrop {
  background: rgba(253, 249, 244, 0.65);
  backdrop-filter: blur(6px);
  -webkit-backdrop-filter: blur(6px);
  border-radius: 12px;
  padding: 16px 20px;
}

/* ── Light text on dark backgrounds (night/moonlit) ── */
.textSchemeLight .overlayPageText {
  color: #f0ece4;
  text-shadow: 0 1px 4px rgba(0, 0, 0, 0.5);
}

.textSchemeLight .overlayTextBackdrop {
  background: rgba(15, 20, 35, 0.55);
  backdrop-filter: blur(6px);
  -webkit-backdrop-filter: blur(6px);
  border-radius: 12px;
  padding: 16px 20px;
}
```

**In `reader-v2.tsx`**, update the overlay rendering (around line 403):

Replace:
```tsx
<div className={`${styles.overlayTextShell} ${overlayZoneClass(currentPage.textZone)}`}>
  <div className={styles.overlayTextGradient} aria-hidden />
  <p className={styles.overlayPageText}>{currentPage.text || ' '}</p>
</div>
```

With:
```tsx
<div className={`${styles.overlayTextShell} ${overlayZoneClass(currentPage.textZone)} ${currentPage.textColorScheme === 'light' ? styles.textSchemeLight : styles.textSchemeDark}`}>
  <div className={styles.overlayTextBackdrop}>
    <p className={styles.overlayPageText}>{currentPage.text || ' '}</p>
  </div>
</div>
```

Remove the `<div className={styles.overlayTextGradient} aria-hidden />` — it's replaced by the backdrop.

**Important CSS change to `.overlayPageText`** (line 228):
Remove the hardcoded `color: #2a241a;` and `text-shadow: none;` — these are now set by the scheme classes above.
