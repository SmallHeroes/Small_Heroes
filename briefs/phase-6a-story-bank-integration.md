# Phase 6a — Story Bank Integration

## Goal
Wire pre-written story-bank markdown files into the generation pipeline so we can render a full book (with illustrations) from any story in `story-bank/raw/` — bypassing all LLM story generation stages (Brain, Outline, Prose) and going directly to image generation.

## Why
We have 30 curated, QA'd stories in `story-bank/raw/`. To see them rendered as real books with illustrations + reader, we need a loader that:
1. Parses the markdown format
2. Builds a `GeneratedStory` object from it
3. Feeds it into the existing image generation pipeline (Stage 4+)
4. Creates DB records (Order, GeneratedBook, BookPage) so the reader can display it

## Architecture

### New Files

#### 1. `backend/providers/story-bank-loader.ts`
Parses a story-bank markdown file and returns a `GeneratedStory`.

```typescript
import type { GeneratedStory, StoryPage, ShotVisualDirection } from './pipeline';
import fs from 'fs/promises';
import path from 'path';

interface StoryBankMeta {
  title: string;
  storyStyle: string;
  metaphor: string;
  stakes: string;
  emotionalArc: string;
  solutionType: string;
}

/**
 * Parse a story-bank markdown file into a GeneratedStory object.
 * Skips all LLM stages — uses imageDirection fields as rawScenePrompt.
 */
export async function loadStoryFromBank(filePath: string, childName: string, companionName: string): Promise<GeneratedStory> {
  const raw = await fs.readFile(filePath, 'utf-8');
  
  // Extract title from first story header: === STORY N: TITLE ===
  const titleMatch = raw.match(/===\s*STORY\s*\d+:\s*(.+?)\s*===/);
  const title = titleMatch?.[1] ?? 'סיפור מהבנק';
  
  // Extract metadata
  const storyStyle = extractField(raw, 'storyStyle');
  const solutionType = extractField(raw, 'solutionType');
  
  // Split into pages by "--- Page N ---"
  const pageBlocks = raw.split(/---\s*Page\s*(\d+)\s*---/).slice(1);
  // pageBlocks is now [pageNum, content, pageNum, content, ...]
  
  const pages: StoryPage[] = [];
  for (let i = 0; i < pageBlocks.length; i += 2) {
    const pageNumber = parseInt(pageBlocks[i], 10);
    const block = pageBlocks[i + 1];
    
    // Extract imageDirection line
    const imageDirectionMatch = block.match(/imageDirection:\s*(.+)/);
    const imageDirection = imageDirectionMatch?.[1]?.trim() ?? '';
    
    // Extract text (everything before imageDirection, trimmed)
    const textPart = block.replace(/imageDirection:.*/, '').trim();
    
    // Replace {{childName}} and {{companionName}} placeholders
    const text = textPart
      .replace(/\{\{childName\}\}/g, childName)
      .replace(/\{\{companionName\}\}/g, companionName);
    
    const resolvedImageDirection = imageDirection
      .replace(/\{\{childName\}\}/g, childName)
      .replace(/\{\{companionName\}\}/g, companionName);

    // Parse imageDirection into ShotVisualDirection
    const visualDirection = parseImageDirection(resolvedImageDirection);
    
    pages.push({
      pageNumber,
      text,
      narrationText: text, // same as text for story-bank stories
      imageSubject: visualDirection?.mainAction ? 'action' : 'child',
      imagePrompt: resolvedImageDirection, // raw English scene description
      rawScenePrompt: resolvedImageDirection,
      visualDirection,
    });
  }
  
  return {
    title,
    coverText: title,
    characterSheet: {
      mainCharacter: { name: childName, visualDescription: '' }, // filled by caller
      supportingCharacters: [{ name: companionName, relationship: 'companion', visualDescription: '' }],
      worldDescription: '',
    },
    concept: {
      centralEntity: { name: companionName, type: 'external_helper', visualDescription: '', behaviorRules: ['', '', ''], strangeDetail: '' },
      narrativePurpose: { represents: '', whyItAppears: '', whatItNeedsOrWants: '' },
      resilienceLayer: { identificationAnchor: '', projectionLogic: '', regulationAction: '', transformationMarker: '' },
      surpriseOrShift: '',
      emotionalPeak: '',
      resolution: { action: '', transformation: '' },
    },
    pages,
    visualBible: null as any, // will be generated by Visual Bible stage
    pageCompositionPlan: null as any, // will be generated
    heroVisualLock: undefined,
    styleLock: undefined,
    entityVisualLock: undefined,
    meta: {
      provider: 'story-bank',
      model: 'pre-written',
      tokens: 0,
      totalTokens: 0,
    },
  };
}

/**
 * Parse an imageDirection string into a ShotVisualDirection.
 * Format: "Description, camera type, focal point: X, lighting"
 */
function parseImageDirection(dir: string): ShotVisualDirection | undefined {
  if (!dir) return undefined;
  
  // Extract focal point
  const focalMatch = dir.match(/focal point:\s*([^,]+)/i);
  const focal = focalMatch?.[1]?.trim() ?? '';
  
  // Extract camera
  const cameraMatch = dir.match(/(wide shot|medium shot|close shot|low angle|bird-eye view|bird's eye)/i);
  const camera = cameraMatch?.[1]?.trim() ?? 'medium shot';
  
  // Extract lighting
  const lightMatch = dir.match(/([\w\s]+lighting|[\w\s]+light|[\w\s]+mood)$/i);
  const lighting = lightMatch?.[1]?.trim() ?? '';
  
  // Main action = everything before the first comma (or camera term)
  const mainAction = dir.split(',')[0]?.trim() ?? dir;
  
  return {
    locationZone: 'garden', // most stories happen in a garden — can be overridden
    mainAction,
    visibleObjects: focal ? [focal] : [],
    characterPose: '',
    emotionVisual: '',
    lightingSource: lighting,
    environmentDetail: '',
    textTranslation: '', // Hebrew text is in the page already
    mustInclude: focal ? [focal] : [],
    mustNotInclude: [],
    camera,
    composition: '',
  };
}

function extractField(raw: string, field: string): string {
  const match = raw.match(new RegExp(`${field}:\\s*(.+)`));
  return match?.[1]?.trim() ?? '';
}
```

#### 2. `app/api/dev/story-bank/route.ts`
Dev-only API endpoint to trigger a story-bank render.

```typescript
import { NextRequest, NextResponse } from 'next/server';
import { prisma } from '@/lib/prisma';
import { loadStoryFromBank } from '@/backend/providers/story-bank-loader';
import { generateAllPageImages, generateBookCover } from '@/backend/providers/image';
import { assignTemplatesForBook } from '@/lib/bookPageLayout';
import {
  buildPresentationWebpFromBuffer,
  evaluateImageSignal,
  fetchImageBuffer,
  placementModeFromPageTemplate,
} from '@/lib/illustrationPresentation';
import { storePresentationBuffer } from '@/lib/image-storage';
import { randomUUID } from 'crypto';
import path from 'path';

export async function POST(req: NextRequest) {
  // Dev-only guard
  if (process.env.NODE_ENV === 'production') {
    return NextResponse.json({ error: 'Not available in production' }, { status: 404 });
  }

  const body = await req.json().catch(() => ({}));
  const {
    storyFile,       // e.g. "batch-05_15b.md"
    childName = 'נועה',
    childGender = 'girl',
    childAge = 5,
    companionName = 'צפרדע',
    illustrationStyle = 'realistic_illustrated',
    childImageUrl = null,
  } = body;

  if (!storyFile) {
    return NextResponse.json({ error: 'storyFile is required' }, { status: 400 });
  }

  const filePath = path.join(process.cwd(), 'story-bank', 'raw', storyFile);

  try {
    // 1. Load story from bank
    const story = await loadStoryFromBank(filePath, childName, companionName);
    console.log(`[StoryBank] Loaded "${story.title}" — ${story.pages.length} pages`);

    // 2. Create a fake order + book records
    const orderId = randomUUID();
    const accessKey = randomUUID().slice(0, 8);

    const order = await prisma.order.create({
      data: {
        id: orderId,
        status: 'generating',
        childName,
        childAge,
        childGender,
        childTraits: [],
        topic: 'story_bank',
        challengeItems: [],
        outcomeItems: [],
        helperItems: [],
        avoidItems: [],
        storyLength: 'medium',
        illustrationStyle,
        childImageUrl,
        accessKey,
        totalPrice: 0,
        textStatus: 'done',
        imageStatus: 'running',
        audioStatus: 'done',
        packageStatus: 'pending',
        bookName: story.title,
        paymentProvider: 'fake',
      },
    });

    const book = await prisma.generatedBook.create({
      data: {
        orderId,
        title: story.title,
        coverText: story.coverText,
      },
    });

    // 3. Create BookPage records (text is already done)
    await prisma.bookPage.createMany({
      data: story.pages.map(p => ({
        bookId: book.id,
        pageNumber: p.pageNumber,
        text: p.text,
        narrationText: p.narrationText,
      })),
    });

    // 4. Generate images using existing pipeline
    const childDesc = `A ${childGender === 'girl' ? 'girl' : 'boy'} named ${childName}, approximately ${childAge} years old, warm and friendly appearance`;

    const imageOutcome = await generateAllPageImages(
      story.pages.map((p, idx) => {
        const prev = idx > 0 ? story.pages[idx - 1] : null;
        return {
          pageNumber: p.pageNumber,
          imagePrompt: p.imagePrompt,
          rawScenePrompt: p.rawScenePrompt,
          visualDirection: p.visualDirection,
          bookPageText: p.text,
          pageIntent: undefined,
          composition: undefined,
          compositionRules: undefined,
          environmentContinuity: prev ? `continue from page ${prev.pageNumber}` : 'opening page',
        };
      }),
      {
        illustrationStyle,
        childName,
        childDescription: childDesc,
        referenceImages: childImageUrl ? [childImageUrl] : undefined,
        orderId,
        heroVisualLock: undefined,
        styleLock: undefined,
        entityVisualLock: undefined,
      }
    );

    // 5. Save image assets
    const dbPages = await prisma.bookPage.findMany({ where: { bookId: book.id } });
    for (const page of dbPages) {
      const image = imageOutcome.results.get(page.pageNumber);
      if (image) {
        await prisma.imageAsset.create({
          data: {
            pageId: page.id,
            provider: image.provider,
            prompt: image.prompt,
            url: image.url,
            rawUrl: image.rawUrl ?? null,
            width: image.width,
            height: image.height,
            style: illustrationStyle,
          },
        });
      }
    }

    // 6. Run presentation layer (template assignment + presentation URLs)
    const pagesWithImages = await prisma.bookPage.findMany({
      where: { bookId: book.id },
      orderBy: { pageNumber: 'asc' },
      include: { imageAsset: true },
    });

    const templateAssignments = assignTemplatesForBook(
      pagesWithImages.map(p => ({
        pageNumber: p.pageNumber,
        text: p.text,
        imageUrl: p.imageAsset?.url ?? null,
      }))
    );

    for (const assignment of templateAssignments) {
      const page = pagesWithImages.find(p => p.pageNumber === assignment.pageNumber);
      if (!page || !page.imageAsset) continue;

      await prisma.bookPage.update({
        where: { id: page.id },
        data: { layoutTemplate: assignment.template },
      });

      // Build presentation image
      try {
        const imgBuffer = await fetchImageBuffer(page.imageAsset.url);
        if (imgBuffer) {
          const signal = evaluateImageSignal(imgBuffer);
          const placement = placementModeFromPageTemplate(assignment.template);
          const presBuffer = await buildPresentationWebpFromBuffer(imgBuffer, {
            placement,
            signal,
          });
          const presUrl = await storePresentationBuffer(presBuffer, orderId, page.pageNumber);
          await prisma.imageAsset.update({
            where: { id: page.imageAsset.id },
            data: { presentationUrl: presUrl },
          });
        }
      } catch (e) {
        console.warn(`[StoryBank] Presentation failed for page ${page.pageNumber}:`, e);
      }
    }

    // 7. Mark order as ready
    await prisma.order.update({
      where: { id: orderId },
      data: {
        status: 'ready',
        imageStatus: 'done',
        packageStatus: 'done',
      },
    });

    // Return the book URL
    const bookUrl = `/read/${accessKey}`;

    return NextResponse.json({
      success: true,
      orderId,
      bookId: book.id,
      accessKey,
      bookUrl,
      pagesRendered: imageOutcome.results.size,
      pagesFailed: imageOutcome.failedPages,
    });

  } catch (error: any) {
    console.error('[StoryBank] Generation failed:', error);
    return NextResponse.json({ error: error.message }, { status: 500 });
  }
}
```

#### 3. `app/dev/story-bank/page.tsx`
Simple dev UI page to trigger a story-bank render.

```tsx
'use client';

import { useState } from 'react';

const AVAILABLE_STORIES = [
  { file: 'batch-05_15b.md', label: '15b — העץ שלא הסכים לזרום (FORCE)' },
  { file: 'batch-05_13a.md', label: '13a — החדר שכמעט נשכח (EXIT)' },
  { file: 'batch-05_13b.md', label: '13b — הקופסה שלא נתנה לעבור (TRICK)' },
  { file: 'batch-05_14a.md', label: '14a — הרוח שלא הפסיקה לרוץ (REDIRECT)' },
  { file: 'batch-05_14b.md', label: '14b — השער שלא נפתח בחינם (TRADE)' },
  { file: 'batch-05_15a.md', label: '15a — הזרעים שלא הסכימו לגדול (RULE_HACK)' },
];

export default function StoryBankDevPage() {
  const [storyFile, setStoryFile] = useState(AVAILABLE_STORIES[0].file);
  const [childName, setChildName] = useState('נועה');
  const [childGender, setChildGender] = useState('girl');
  const [companionName, setCompanionName] = useState('צפרדע');
  const [illustrationStyle, setIllustrationStyle] = useState('realistic_illustrated');
  const [status, setStatus] = useState<'idle' | 'loading' | 'done' | 'error'>('idle');
  const [result, setResult] = useState<any>(null);

  async function handleGenerate() {
    setStatus('loading');
    setResult(null);
    try {
      const res = await fetch('/api/dev/story-bank', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ storyFile, childName, childGender, companionName, illustrationStyle }),
      });
      const data = await res.json();
      if (!res.ok) throw new Error(data.error || 'Failed');
      setResult(data);
      setStatus('done');
    } catch (e: any) {
      setResult({ error: e.message });
      setStatus('error');
    }
  }

  return (
    <div style={{ padding: 40, maxWidth: 600, margin: '0 auto', fontFamily: 'sans-serif' }}>
      <h1>Story Bank — Dev Renderer</h1>
      <p style={{ color: '#666' }}>Render a pre-written story with illustrations. Dev only.</p>

      <div style={{ display: 'flex', flexDirection: 'column', gap: 16, marginTop: 24 }}>
        <label>
          Story:
          <select value={storyFile} onChange={e => setStoryFile(e.target.value)} style={{ display: 'block', width: '100%', padding: 8, marginTop: 4 }}>
            {AVAILABLE_STORIES.map(s => (
              <option key={s.file} value={s.file}>{s.label}</option>
            ))}
          </select>
        </label>

        <label>
          Child Name:
          <input value={childName} onChange={e => setChildName(e.target.value)} style={{ display: 'block', width: '100%', padding: 8, marginTop: 4 }} />
        </label>

        <label>
          Gender:
          <select value={childGender} onChange={e => setChildGender(e.target.value)} style={{ display: 'block', width: '100%', padding: 8, marginTop: 4 }}>
            <option value="girl">Girl</option>
            <option value="boy">Boy</option>
          </select>
        </label>

        <label>
          Companion Name:
          <input value={companionName} onChange={e => setCompanionName(e.target.value)} style={{ display: 'block', width: '100%', padding: 8, marginTop: 4 }} />
        </label>

        <label>
          Style:
          <select value={illustrationStyle} onChange={e => setIllustrationStyle(e.target.value)} style={{ display: 'block', width: '100%', padding: 8, marginTop: 4 }}>
            <option value="realistic_illustrated">Realistic Illustrated (Style 01)</option>
            <option value="pencil_watercolor">Pencil Watercolor (Style 02)</option>
          </select>
        </label>

        <button
          onClick={handleGenerate}
          disabled={status === 'loading'}
          style={{ padding: '12px 24px', fontSize: 16, background: '#7c3aed', color: '#fff', border: 'none', borderRadius: 8, cursor: 'pointer', opacity: status === 'loading' ? 0.5 : 1 }}
        >
          {status === 'loading' ? 'Generating... (2-5 min)' : 'Generate Book'}
        </button>
      </div>

      {result && (
        <div style={{ marginTop: 24, padding: 16, background: status === 'error' ? '#fef2f2' : '#f0fdf4', borderRadius: 8 }}>
          {status === 'done' && (
            <>
              <p style={{ color: '#16a34a', fontWeight: 'bold' }}>✓ Book ready!</p>
              <p>Pages rendered: {result.pagesRendered} / {result.pagesRendered + (result.pagesFailed?.length || 0)}</p>
              {result.pagesFailed?.length > 0 && <p style={{ color: '#dc2626' }}>Failed: pages {result.pagesFailed.join(', ')}</p>}
              <a href={result.bookUrl} target="_blank" style={{ display: 'inline-block', marginTop: 8, padding: '8px 16px', background: '#7c3aed', color: '#fff', borderRadius: 6, textDecoration: 'none' }}>
                Open Book →
              </a>
            </>
          )}
          {status === 'error' && <p style={{ color: '#dc2626' }}>{result.error}</p>}
        </div>
      )}
    </div>
  );
}
```

## Key Design Decisions

1. **No Visual Bible stage** — story-bank stories use `imageDirection` as the scene description directly. The GPT Image provider will use this as `rawScenePrompt` which feeds into `buildGPTImagePrompt`. This means images won't have the full Visual Bible consistency locks, but it's fine for testing. Future: we can add a mini Visual Bible generation stage that takes the full story text and generates character/style locks before image gen.

2. **No composition plan** — Camera angles are extracted from imageDirection strings (e.g., "wide shot", "close shot", "low angle"). This is sufficient for GPT Image.

3. **Dev-only** — Behind `NODE_ENV !== 'production'` guard. No payment needed.

4. **Same image pipeline** — Uses the exact same `generateAllPageImages` + presentation layer as production. What you see is what production would produce.

5. **Template placeholders** — `{{childName}}` and `{{companionName}}` are replaced at load time, so the same story works for any child.
