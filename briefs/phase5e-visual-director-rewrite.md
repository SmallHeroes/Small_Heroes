# Phase 5e — Visual Director Rewrite (Scene-as-Direction, Not Scene-as-Poetry)

## Problem

Phase 5d fixed the technical pipeline (killed `images.edit`, shortened prompts, scene-first order). But the images are STILL too similar because:

1. **The storyboard LLM writes poetry, not image directions** — "A lone glowing point rests quietly at the edge of the shelf" tells GPT Image nothing concrete to draw
2. **All scenes are in the same location** — night-fear stories produce 8 bedroom scenes with "glowing dots/lines" on every page
3. **Composition info dissolves into prose** — the LLM gets a detailed PageComposition plan but melts it into flowery language that gets trimmed
4. **No emotional arc in visuals** — emotion is expressed as atmosphere words, not physical poses/actions
5. **Cover is broken** — uses legacy `buildCoverPrompt` with Hebrew text and dark scene description

## Root Cause

The current `buildShotPlanPrompt` asks the LLM to write 4-5 sentences of "illustrator-to-illustrator" prose. This produces beautiful writing but terrible image prompts. GPT Image needs:

- **WHO** is in the scene (concrete description)
- **WHERE** exactly (specific location zone, not "a room")
- **WHAT** they're physically doing (action verb, not mood)
- **WHAT OBJECTS** are visible
- **HOW** framed (camera, scale, position in frame)

It does NOT need: metaphors, poetic hooks, atmosphere words, style descriptions (we add those separately).

## Architecture Decision

**We already have `PageComposition`** generated by a separate LLM call — it has camera distance, angle, hero placement, entity placement, composition type, etc. The shot plan LLM gets this as INPUT but wastes it by dissolving it into prose.

**New approach: The shot plan LLM outputs STRUCTURED JSON, and `buildGPTImagePrompt` assembles the final prompt mechanically.**

This gives us:
- Deterministic camera rotation (from PageComposition)
- Concrete scene descriptions (from structured JSON)
- Consistent character block (added mechanically)
- Short, focused style block (added mechanically)

### Change 2: Rewrite `buildGPTImagePrompt` to use structured data

```typescript
function buildGPTImagePrompt(input: ImageInput): string {
  const isPreview = !!input.isDirectionPreview;

  // ── SCENE (~250-350 chars, concrete physical description) ──
  const rawScene = (input.rawScenePrompt ?? '').trim();
  const stage4 = (input.stage4Prompt ?? '').trim();
  const fallbackScene = extractSceneCore(input.pagePrompt || '').trim();
  const sceneDesc = rawScene || stage4 || fallbackScene || (input.bookPageText ?? '').trim() || '';
  
  // Trim to max 400 chars (down from 600)
  const trimmedScene = sceneDesc.length > 400
    ? sceneDesc.slice(0, 397).replace(/\s\S*$/, '...')
    : sceneDesc;

  // ── CHARACTER DNA (once, short, ~150 chars) ──
  const charParts: string[] = [];
  if (input.childDescription) {
    charParts.push(`Main character: ${input.childDescription}`);
  }
  if (input.companion) {
    const comp = input.companion;
    charParts.push(`Companion: ${comp.name}, ${comp.visualDescription}`);
  }
  const characterBlock = charParts.length > 0 ? charParts.join('\n') : '';

  // ── COMPOSITION (mechanical, from PageComposition — ~100 chars) ──
  const compParts: string[] = [];
  
  // Camera instruction from composition plan or mechanical rotation
  if (input.pageNumber && input.totalPages) {
    const cameraRotation = [
      'Wide shot, character small in the scene.',          // page 1
      'Medium shot, character fills lower third.',          // page 2
      'Close-up, focus on hands or face.',                  // page 3
      'Over-shoulder view from behind character.',          // page 4
      'Wide shot from a different angle than page 1.',      // page 5
      'Close-up of companion or key object.',               // page 6
      'Dynamic angle, slightly above looking down.',        // page 7
      'Calm wide shot, peaceful composition.',              // page 8
    ];
    const camIdx = ((input.pageNumber - 1) % cameraRotation.length);
    compParts.push(cameraRotation[camIdx]);
    compParts.push(`Page ${input.pageNumber} of ${input.totalPages} — must look completely different from other pages.`);
  }
  
  if (input.textZone) {
    const tzMap: Record<string, string> = {
      top_clear: 'Top 40% must be empty light space for text overlay.',
      bottom_clear: 'Bottom 30% must be open for text.',
      left_clear: 'Left 30% open for text.',
      right_clear: 'Right 30% open for text.',
      center_clear: 'Center area open for text.',
    };
    const tzHint = tzMap[input.textZone];
    if (tzHint) compParts.push(tzHint);
  }
  const compositionBlock = compParts.join(' ');

  // ── STYLE (short — ~200 chars) ──
  const styleBlock = isPreview
    ? 'Soft Pixar-style watercolor illustration, light cream background, gentle warm lighting, modern children\'s book. No text or letters.'
    : `Soft Pixar-watercolor children's book illustration. Light cream watercolor background. Soft natural lighting, cheerful. No text, no letters, no UI.`;

  // ── ASSEMBLE: Scene → Character → Composition → Style ──
  const parts = [trimmedScene, characterBlock, compositionBlock, styleBlock].filter(Boolean);
  const fullPrompt = parts.join('\n\n');

  console.log(
    `[gpt_prompt_v4] page=${input.pageNumber} rawScene=${rawScene.length} stage4=${stage4.length} sceneUsed=${trimmedScene.length} final=${fullPrompt.length}`
  );
  console.log(`[gpt_scene] page=${input.pageNumber} "${trimmedScene.slice(0, 200)}"`);
  return fullPrompt;
}
```

**Key changes:**
- Scene cap reduced from 600 → 400 chars
- Mechanical camera rotation array (8 slots, cycling)
- Style block shortened further to ~100 chars (from ~300)
- Target total: ~800-1000 chars (down from ~1400)

```typescript
function buildGPTCoverPrompt(input: CoverImageInput): string {
  // Cover ALWAYS shows happy, inviting scene — NEVER the problem/fear
  const childDesc = input.childDescription || 'a young child';
  const entityDesc = input.entityVisualLock 
    ? `a small friendly ${input.entityVisualLock.shape}, ${input.entityVisualLock.color}`
    : '';
  
  const sceneParts = [
    `A smiling ${childDesc} in a cozy, well-lit room.`,
    entityDesc ? `${entityDesc} sitting close beside them, looking friendly and warm.` : '',
    'The scene feels safe, magical, and inviting.',
    'Bright warm lighting, cheerful atmosphere.',
    'Large empty area at top (25-35%) for title text.',
  ];
  
  const scene = sceneParts.filter(Boolean).join(' ');
  
  const style = `Soft Pixar-watercolor children's book cover illustration. Light cream watercolor background. Cheerful and bright. No text, no letters, no title, no words.`;
  
  return `${scene}\n\n${style}`;
}
```

| Scene description style | Poetic prose | Concrete physical descriptions |
| Scene length | ~600 chars | ~250-350 chars |
| Camera control | "Use a unique composition" (vague) | Mechanical rotation: wide→medium→close→over-shoulder→... |

| Visual motif repetition | "Glowing dots" on 6/8 pages | Max 2 pages per motif |

## Risk: LLM Compliance

The biggest risk is that the LLM ignores the structured format and reverts to prose. Mitigations:
- Strong system prompt with explicit examples of good vs bad
- JSON schema enforcement in the output format
- Post-processing validation: if `visualDirection` is missing, log a warning
- The mechanical camera rotation and style block don't depend on LLM compliance — they're added by code

## Risk: Scene Too Short

350 chars is much less than 750. But GPT Image responds BETTER to short, concrete prompts than long, poetic ones. "The girl kneels beside her bed reaching under it while a fox watches from the shelf" is 80 chars and paints a clearer picture than 750 chars of "glowing points scatter across restless paths."
