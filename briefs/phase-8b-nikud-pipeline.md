# Phase 8b — Add Nikud (Vowel Marks) to Story Text

## Problem

Hebrew story text is displayed without nikud (ניקוד / vowel marks).  
For a children's book product, this is a significant gap — young readers (ages 3-7) need nikud to read Hebrew.

## Solution

Add a nikud post-processing step after story text is finalized, before it's saved to DB.

## Where to Add

### 1. Story-bank path (`app/api/dev/story-bank/route.ts`)

After the story pages are loaded from the story-bank file and before they're passed to image generation, add a nikud step.

**However** — if we run the batch script `scripts/add-nikud.mjs` on all story-bank files first, then the story-bank path already has nikud'd text. In that case, no code change needed here.

### 2. Dynamic generation path (`app/api/generate/route.ts`)

After story is generated by GPT (~line 660, after `storyResult` is ready), add:

```ts
// Add nikud to story text for children's readability
const nikudPages = await addNikudToStory(storyResult.pages);
storyResult.pages = nikudPages;
```

### 3. New utility function

**File:** `backend/providers/nikud.ts` (NEW)

```ts
import OpenAI from 'openai';

const openai = new OpenAI();

export async function addNikudToStory(
  pages: Array<{ text: string; [key: string]: any }>
): Promise<Array<{ text: string; [key: string]: any }>> {
  // Collect all Hebrew text
  const pageTexts = pages
    .map((p, i) => `=== עמוד ${i + 1} ===\n${p.text}`)
    .join('\n\n');

  const response = await openai.chat.completions.create({
    model: 'gpt-4o',
    messages: [
      {
        role: 'system',
        content: `You are a Hebrew nikud expert for children's literature.
Add full nikud (ניקוד מלא) to every Hebrew word.
RULES:
1. Keep the EXACT same text — do not change any word
2. Keep {{childName}} and {{companionName}} as-is
3. Keep all punctuation and line breaks identical
4. Use correct grammatical nikud with proper dagesh
5. Return ONLY the nikud'd text in the same format`
      },
      {
        role: 'user',
        content: `Add full nikud:\n\n${pageTexts}`
      }
    ],
    temperature: 0.1,
    max_tokens: 8000,
  });

  const reply = response.choices[0].message.content || '';
  
  // Parse reply
  const nikudMap: Record<number, string> = {};
  const regex = /===\s*עמ[וּ]ד\s*(\d+)\s*===\s*\n([\s\S]*?)(?====\s*עמ[וּ]ד\s*\d+\s*===|$)/g;
  let m;
  while ((m = regex.exec(reply)) !== null) {
    nikudMap[parseInt(m[1])] = m[2].trim();
  }

  // Apply nikud to pages, falling back to original if missing
  return pages.map((p, i) => ({
    ...p,
    text: nikudMap[i + 1] || p.text,
  }));
}
```

### 4. Wrap with try/catch

Nikud is a nice-to-have, not critical. If the API call fails, the book should still generate with plain text:

```ts
try {
  storyResult.pages = await addNikudToStory(storyResult.pages);
} catch (err) {
  console.warn('Nikud step failed, continuing without:', err.message);
}
```
